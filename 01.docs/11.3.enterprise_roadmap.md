# 11.3 Enterprise Intelligence — 로드맵(도메인 관점)

> 본 문서는 Enterprise Intelligence 도메인 기능 확장의 로드맵과 “완료 기준(테스트/문서)”을 정의합니다.

---

## 1. Phase A (현재 정리): 기존 3에이전트 통합 기반 확립

목표:
- Search RAG / Presentation / Prior Art(KIPRIS)의 연계 비용을 낮추는 공용 규약 정착

완료 기준:
- AgentCatalog 기반 worker discovery가 안정적으로 동작
- 최소 기능 경로(채팅→검색→근거→응답, 채팅→PPT)가 functional 테스트로 고정
- 10/11/21 문서가 동일한 베이스라인을 설명

---

## 2. Phase B: Text-to-SQL 통합(정형 지식 결합)

목표:
- SQL 기반 근거를 Evidence로 통합

완료 기준:
- Schema RAG + SQL Guards + read-only 우선
- unit/functional/integration 게이트 통과
- 하이브리드 라우팅(rag/sql/hybrid) 시나리오가 functional 테스트로 고정

---

## 3. Phase C: Deep Research 통합(외부 연결)

목표:
- 조사/검증/인용 중심의 리서치 오케스트레이션

완료 기준:
- snippet-first/모델 라우팅/캐싱 전략이 설계/구현에 포함
- 비용/토큰 폭증을 제어하는 가드레일 존재

---

## 4. Phase D: 분석 기능 확장(Topic/Network/KG)

목표:
- 분석 기능을 플러그인처럼 추가할 수 있는 capability 기반 확장

완료 기준:
- AgentCatalog가 capability discovery를 제공(또는 동등한 방식)
- Analysis Team이 최소 1개 분석 워커를 포함하고 E2E 최소 경로가 검증됨
