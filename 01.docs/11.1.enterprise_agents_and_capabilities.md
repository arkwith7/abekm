# 11.1 Enterprise Intelligence — 에이전트 라인업 & Capability

> 본 문서는 Enterprise Intelligence(IP/연구/시장 인텔리전스) 도메인에서 제공할 에이전트 라인업과 capability 기준을 정의합니다.
> 공용 프레임워크/오케스트레이션/계약/테스트 규칙은 10.x 문서를 참조합니다.

참조:
- 공용 프레임워크 인덱스: [01.docs/10.agentic_architecture.md](10.agentic_architecture.md)
- 개선 계획: [01.docs/21.agentic_ai_refactor_plan.md](21.agentic_ai_refactor_plan.md)

---

## 1. 도메인 목표

- 특허/논문/시장 데이터를 결합하여 “검색”을 넘어 **인사이트 생성(분석/예측/제언)** 까지 확장
- 문서 RAG + Text-to-SQL + 외부 연결(Deep Research)로 정형/비정형 지식을 결합

---

## 2. 에이전트 그룹(Team-of-Teams 기준)

권장 팀 구성:
- **Research Team**: Search RAG, Prior Art(KIPRIS), Text-to-SQL, Deep Research
- **Analysis Team**: Topic Modeling, Network Analysis, Knowledge Graph(및 Graph RAG)
- **Output Team**: Presentation(PPT), Report

---

## 3. 현재 구현/운영 중(정리 대상)

> 실제 코드 경로는 점진적으로 feature-pack으로 이행하며, Supervisor는 AgentCatalog에서 workers를 조회합니다.

- **Search RAG (실시간 채팅 RAG)**
  - 현재 코드: `backend/app/agents/paper_search_agent.py`
  - 주요 도구: `vector_search`, `keyword_search`, `fulltext_search`, `rerank`, `deduplicate`, `context_builder`, (필요 시) `internet_search`

- **Presentation (RAG 연동 PPT 생성)**
  - 현재 코드: `backend/app/agents/presentation/` 및 `backend/app/services/presentation/`
  - 주요 도구: `generate_outline`, `create_slides` (또는 presentation toolset)

- **Prior Art (KIPRIS 기반 선행기술 검색)**
  - 현재 코드: `backend/app/tools/retrieval/patent_search_tool.py` 등 + 특허 분석 도구
  - 방향: “도구 중심”에서 “에이전트(Worker) 중심”으로 승격하여 통합

---

## 4. 추가 통합 예정(우선순위)

- **Text-to-SQL Agent**
  - 목표: 통계/수치/집계/정형 질의의 신뢰성 있는 답변
  - 필수: Schema RAG + SQL Guards + read-only 우선

- **Deep Research Agent**
  - 목표: 문서 RAG + SQL + 웹/외부 소스까지 결합한 조사/리서치 오케스트레이션
  - 운영: snippet-first, 모델 라우팅, 캐싱을 초기부터 설계

---

## 5. 분석/예측 확장(플러그인화)

분석 기능(토픽 모델링/네트워크/지식그래프)은 초기에는
- Tool(분석 함수) + 1개의 Worker(분석 오케스트레이션)
로 시작하고, 성숙하면 전용 subgraph로 승격합니다.

Registry/Capability discovery는 새로운 레지스트리를 만들기보다, `AgentCatalog` 확장으로 흡수하는 것을 권장합니다.
