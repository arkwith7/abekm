# 21.1 Agentic AI 로드맵 (향후 계획 단일 소스)

> 목적: ABEKM의 Agentic AI 멀티에이전트 플랫폼 확장을 위한 **향후 계획/우선순위/완료 기준**을 단일 소스로 관리합니다.
>
> 현재 구현된 기준선(완료 기록)은 [01.docs/21.agentic_ai_refactor_plan.md](21.agentic_ai_refactor_plan.md)를 참고하세요.

---

## 0. 현재 기준선(2026-01-03)

- Supervisor → AgentCatalog → feature-pack worker discovery 경로가 활성화됨
- 활성 워커(기본): `SearchAgent` / `PresentationAgent` / `PriorArtAgent`
- 프롬프트는 feature-pack 내부 `prompts/`로 로컬화됨
- 중앙 `app/tools/`는 제거되고, 도구는 feature-pack 소유권 기준으로 이관됨

---

## 1. Phase B: Text-to-SQL 통합(정형 지식 결합)

- 설계 문서: [01.docs/14.text_to_sql_agent_design.md](14.text_to_sql_agent_design.md)

목표:
- 정형 데이터 질의(SQL)를 “근거(Evidence)로 합성 가능한 형태”로 안정적으로 제공

핵심 구성(권장):
- Schema RAG(테이블/컬럼 메타데이터 컨텍스트)
- Few-shot 예제
- SQL Guards(allowlist, DDL/DML 차단, WHERE/limit 검사)
- Read-only 우선

완료 기준:
- `rag/sql/hybrid` 라우팅 시나리오가 functional 테스트로 고정됨
- unit/functional/integration 게이트 통과

---

## 2. Phase C: Deep Research 확장(외부 연결 포함)

목표:
- 계획 수립 → 소스별 조사(RAG/SQL/web/patent) → 근거 정리 → 합성 보고서

운영 가드레일:
- snippet-first
- 모델 라우팅(저비용/고성능 분리)
- 시맨틱 캐싱(중간 산출물 재사용)

완료 기준:
- 비용/토큰 폭증을 제어하는 가드레일이 코드/테스트/문서에 존재
- 최소 성공 경로가 functional 테스트로 고정

---

## 3. Phase D: Team-of-Teams(2단계 Supervisor) 도입

목표:
- 워커 수 증가 시 단일 Supervisor 비대화를 방지

구성(권장):
- Top-Level Supervisor: 팀 라우팅
- Team Supervisor: 팀 내부 워커 라우팅

완료 기준:
- Top-Level → Team Supervisor → Worker 최소 성공 경로가 functional 테스트로 검증

---

## 4. Patent 데이터 소스 확장(USPTO/EPO/JPO/CNIPA)

목표:
- 데이터 소스 확장 시 구조 변경이 최소화되는 “클라이언트 추가형” 확장

완료 기준:
- 각 소스별 client 추가 후, 통합 검색(aggregator) 경로로 정상 동작
- (가능한 범위에서) 외부 의존성은 테스트 더블/스킵 기준이 명확

---

## 5. 문서/변경관리 규칙

- 코드 변경 단위마다 unit → functional → integration 게이트를 유지
- 기준선 문서는 10.x/11.x에 반영
- 완료된 작업은 [01.docs/21.agentic_ai_refactor_plan.md](21.agentic_ai_refactor_plan.md)에 누적 기록
